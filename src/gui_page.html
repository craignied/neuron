neuron

1 — Dataset
inputs are read from the file — neuron uses a single output
a matched, already-split held-out set — e.g. BP40test.txt
2 — Model
3 — Train
Train picks up from the current weights, so clicking it again continues for more iterations. Randomize weights (with the seed, if set) to start over from a fresh random point.
4 — Stepwise regression
Groups of input nodes make one variable: semicolons separate variables, commas join nodes, hyphens give ranges. The network must be trained first; the regression report is written to neuron.log beside your data.
5 — Session files
Each file is written into the working directory where neuron was started, and downloaded by your browser. Network + scaling factors are what tools/neuron2web.py needs to deploy the model; training/test sets and guesses document the run.

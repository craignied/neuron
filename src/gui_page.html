Covariate stratification — when not to

The outcome is always balanced — that part is free. Balancing on covariates too is a judgment call, not a free upgrade. Leave it off when:

  • you want the holdout to detect covariate drift — matching the test set to the training mix hides the very shift a held-out set exists to catch;
  • you want an independent estimate, not a balanced one (matching makes train and test more alike than two fresh samples, narrowing apparent variance);
  • n is large — the outcome-only split already reproduces the covariate mix to a few parts per thousand, so this buys a rounding-level guarantee, not real precision;
  • it would fragment the data — too many columns or bins make thin cells that round to zero test rows (watch the diagnostic's “too small to place any test exemplar” line).

Use it when you will report performance within a subgroup, or the data are small enough that a random draw could imbalance a strong predictor.

Balancing makes the test resemble your sample; for a harder test — new patients from sites the model never trained on — use a group-aware split instead.
Group on (keep clusters intact) — blank = none:

Group-aware split — a harder test

Rows that share identical values on the named columns form a cluster (patients in a hospital, a county) that is kept entirely on one side of the split — no cluster straddles train and test.

This answers a different question from stratification: not “how does the model do on new patients from known sites?” but “how does it do on patients from sites it never trained on?” — usually a lower, more honest number when records cluster.

Because clusters are indivisible the achieved test size only approximates your target, and the outcome balance is preserved only as far as whole clusters allow — the diagnostic reports what was achieved. Grouping takes precedence over covariate stratification.
